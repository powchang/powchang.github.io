Portfolio Natural Language Porcessing

Few Short Meta Learning AI Conversational

Goal-oriented dialogue systems is a category of dialogue systems designed to solve one or multiple specific goals or tasks (e.g. flight reservation, hotel reservation, food ordering, appointment scheduling). Traditionally, goal-oriented dialogue systems are set up as a pipeline with four main modules: 1-Natural Language Understanding (NLU), 2-Dialogue State Tracking (DST), 3-Dialog Policy Manager, and 4-Response Generator. NLU extracts the semantic information from each dialogue turn which includes e.g. user intents and slot values mentioned by the user or system. DST takes the extracted entities to build the state of the user goal by aggregating and tracking the information across all turns of the dialogue. Dialog Policy Manager is responsible for deciding the next action of the system based on the current state. Finally, Response Generator converts the system action into human natural text understandable by the user.

Multi-Layer Perceptron Neural Network for Improving Detection Performance of Malicious Phishing URLs Without Affecting Other Attack Types Classification

The hypothesis here states that neural network algorithms such as Multi-layer Perceptron (MLP) have higher accuracy in differentiating malicious and semi-structured phishing URLs. Flight Delay Time-Series Modeling

Using Machine Learning and Deep Learning Model to Improve Delay Prediction

There are several technical approaches that could potentially be used to improve flight delay prediction using machine learning: Use a larger and more comprehensive dataset: A larger dataset that includes a wider range of variables and more complete information about past flights and the conditions that led to delays could potentially improve the accuracy of the prediction. Use advanced machine learning techniques: More advanced machine learning techniques, such as deep learning or ensembles of multiple models, may be able to better capture the complex relationships between different factors that can cause delays. Incorporate real-time data: Incorporating real-time data, such as current weather conditions and aircraft maintenance status, could potentially improve the accuracy of the prediction by taking into account factors that are changing dynamically. Use domain knowledge: Incorporating domain knowledge about the factors that typically cause delays, such as the types of maintenance issues that are most likely to lead to delays, could potentially improve the accuracy of the prediction. Evaluate and optimize model performance: Careful evaluation and optimization of the performance of the machine learning model, including the use of appropriate evaluation metrics and the selection of hyperparameters, could potentially improve the accuracy of the prediction. Overall, a combination of these approaches may be necessary to achieve significant improvements in flight delay prediction using machine learning.

Patents and Awards

Current controlled multi-state parallel test for semiconductor device Current

Credential ID US US6381718B1

A semiconductor memory device (300) having a parallel test circuit is disclosed. A test data path (308) receives parallel I/O line (I/O0-I/O7) values, and generates therefrom test result data values (PASS and DATA_TST). The test result data values (PASS and DATA_TST) are coupled to a gate control circuit (312). The gate control circuit (312) provides either a first logic value, a second logic value, or an intermediate logic value to an open drain output driver (314) depending upon the test result data values (PASS and DATA_TST). In response to the logic values received from the gate control circuit (312), the open drain output driver (314) drives a data output (DQ) to a first, second or intermediate logic level.

Two pass multi-state parallel test for semiconductor device

Credential ID US US6408411B1

A semiconductor memory device (300) having a parallel test circuit is disclosed. A test data path receives parallel I/O line (I/O0-I/O7) values, and generates therefrom test result data values (PASS and DATA_TST). The test result data values (PASS and DATA_TST) are coupled to a two-bit register (312) and output in a sequential fashion to an open drain output driver (314). In this manner, test result data values are provided by driving an output (DQ) in a rapid sequential fashion, rather than placing the output at one of three states (such as logic high state, a logic low state, or a high impedance state).
